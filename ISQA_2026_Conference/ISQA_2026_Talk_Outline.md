#### Slide 4: The Dreyfus Model
```
🧠 The Science of Skill Acquisition

   Dreyfus Model (1980) - How humans develop expertise
   
   ┌─────────────────────────────────────────────┐
   │  5. EXPERT      │ Intuitive, innovates      │
   │  4. PROFICIENT  │ Sees big picture          │
   │  3. COMPETENT   │ Plans & prioritizes       │
   │  2. ADV BEGINNER│ Recognizes patterns       │
   │  1. NOVICE      │ Follows rules             │
   └─────────────────────────────────────────────┘
   
   → Used in nursing, aviation, military... and now QA!
```

---

#### Slide 5: Dreyfus in QA Context
```
📈 What Each Level Looks Like in QA

   NOVICE        "I follow the test case step by step"
   ADV BEGINNER  "I can write test cases for my feature"
   COMPETENT     "I own quality E2E, coordinate with devs"
   PROFICIENT    "I mentor others, design test strategies"
   EXPERT        "I define standards, consult on critical decisions"
   
   💡 Key insight: You can be Level 4 in one skill, 
      Level 2 in another. It's not one overall level!
```

#### Slide 14: Critical Thinking with AI
```
⚠️ When NOT to Trust AI

   AI can HALLUCINATE - always verify!
   
   ❌ Don't blindly copy AI-generated test cases
   ❌ Don't trust AI for security/compliance decisions
   ❌ Don't let AI replace your domain knowledge
   
   ✅ Use AI as a DRAFT, then apply your expertise
   ✅ Verify AI outputs against requirements
   ✅ Use AI to augment, not replace, your thinking
   
   "AI is your copilot, not your autopilot"
```

---

#### Slide 15: The New QA Skill - Prompt Engineering
```
📝 The New QA Skill: Prompt Engineering

   Bad prompt:
   "Write test cases"
   
   Good prompt:
   "Write 5 test cases for user registration feature.
    Include: email validation, password rules, 
    duplicate account handling. Format as Given-When-Then.
    Consider edge cases for Indonesian phone numbers."
   
   → Better prompts = Better AI outputs
   → This is a LEARNABLE skill!
```
